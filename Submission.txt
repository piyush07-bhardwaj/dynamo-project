STEP 1: SUMMARY OF DEFECTS
================================================================================

Four axes of broken Harbor format:

1. FORMAT (task.toml)
   - artifacts was a STRING ("/app/out.json") instead of ARRAY (should be ["/app/report.json"])
   - Wrong artifact path: declares /app/out.json but task produces /app/report.json
   - allow_internet = true, but task is offline and needs reproducibility
   WHY IT BREAKS: Breaks contract between what Harbor expects to collect and what task actually produces.
   Harbor will fail to locate the declared artifact after the task runs.

2. ENVIRONMENT (environment/Dockerfile)
   - Base image NOT PINNED: uses "python:latest" instead of pinned @sha256 digest
   - SOLUTION LEAKED: Dockerfile copies environment/solution_hint.py into the image
   WHY IT BREAKS: 
     a) "latest" is non-reproducible; different build dates pull different Python versions
     b) Copying the reference solution into the agent image gives away the answer
        (agent can just execute the preloaded solution, defeating the task)

3. VERIFIER (tests/test_outputs.py + tests/test.sh)
   - Tests only assert file EXISTS and is NONEMPTY, never validate actual contents
   - Original test_report_exists() passes if any file named report.json is present
   - Original test_report_nonempty() passes if file has any bytes at all
   WHY IT BREAKS: Verifier is GAMEABLE. A no-op agent could pass if a stale file exists.
     An agent could write "hello" and pass. No validation that JSON is valid or values are correct.
   - test.sh writes reward to wrong path (/app/reward.txt instead of /logs/verifier/reward.txt)
   - test.sh does not write ctrf.json, breaking Harbor's test result format

4. INSTRUCTION (instruction.md)
   - Does not state absolute output path (/app/report.json)
   - Does not specify output format (JSON)
   - Does not list numbered success criteria
   - Vague: "save your findings" doesn't constrain the schema
   WHY IT BREAKS: Inconsistent with verifier logic. Agent doesn't know exactly what's expected.
     Instruction says "save findings" but verifier checks for exact JSON schema.

FOUR-WAY CONSISTENCY BROKEN:
   instruction.md (vague) ≠ task.toml artifacts ("/app/out.json" ≠ "/app/report.json") ≠ 
   verifier (checks for /app/report.json) ≠ solve.sh (writes /app/report.json)

import json
from pathlib import Path


def test_report_is_valid_json_object():
    """Success criterion 1: write a valid JSON object to /app/report.json."""
    report_path = Path("/app/report.json")
    assert report_path.exists(), "report.json was not created"

    report = json.loads(report_path.read_text())
    assert isinstance(report, dict), "report.json must contain a JSON object"
    assert set(report) == {"total_requests", "unique_ips", "top_path"}, (
        "report.json must contain total_requests, unique_ips, and top_path"
    )


def test_report_matches_access_log():
    """Success criterion 2: the JSON summary must accurately reflect /app/access.log."""
    report = json.loads(Path("/app/report.json").read_text())
    assert report == {
        "total_requests": 6,
        "unique_ips": 3,
        "top_path": "/index.html",
    }
